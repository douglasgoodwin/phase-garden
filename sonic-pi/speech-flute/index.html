Speech Oboe (Sonic Pi)

Offline pitch replay of spoken voice as sampled oboe

Back to Sonic Pi

The pitch contour of speech is extracted using aubio pitch and replayed through a sampled oboe in Sonic Pi. Each detected pitch in the voice becomes a pitch-shifted oboe tone, following the natural melody of speech — the rises of questions, the falls of statements, the rhythm of pauses.

Disconnected from OSC bridge

Preparing pitch data

The analysis script extracts pitches from a WAV file using aubio, applies median smoothing and octave correction, then groups frames into notes. Install aubio first (brew install aubio), then: 

node sonic-pi/speech-flute-analyze.mjs samples/yourfile.wav

This writes sonic-pi/speech-flute-pitch.json — an array of { time, freq, duration } entries. The JSON is committed so the piece works without aubio installed.

Changing the instrument sample

The Sonic Pi script pitch-shifts a single sustained note to match each detected frequency. Update the oboe path at the top of speech-flute.rb to swap instruments — any mono WAV or AIF of a held note at a known pitch will work. Adjust the reference MIDI note (default 69 = A4) to match your sample.

Playing

  1. Update the sample path D in sonic-pi/speech-flute.rb to match your system
  2. Open Sonic Pi and paste sonic-pi/speech-flute.rb, hit Run
  3. Run npm run bridge in a terminal
  4. Click Start above
